# 《革命街没有尽头》当前设计契约

本文是剧情数据、交互实现、界面文案和自动化验收共用的基准。开发版本号仅用于文件追踪，不应出现在玩家标题页或叙事规则里。

## 1. 核心叙事契约

> 她被处分、离开、结婚与重逢已经发生。你不能改写这些端点，但能决定通往它们的三个动作，以及它们最后如何被记住。

固定的宏观端点是：莱拉遭到处分、决定离开、与卡姆兰结婚、十三年后与阿拉什重逢，并在路口再次分开。玩家书写的是微观动作：她如何表达、如何处理证据、如何告别，以及照片、邮件与视线如何参与记忆。

因此，界面和旁白不得再声称“所有历史完全不可改变”。准确说法是“宏观端点锁定，微观动作由玩家书写”。玩家做过的动作必须通过即时反馈、近期后效、远期回响和结尾镜头被承认；未发生的动作不得出现在结局里。

## 2. 六次关键输入

### 2.1 三次主动作

主动作仍可使用并列卡片，但选项先描述可理解的行为，不向首轮玩家展示隐藏轴或精确分数。

| 场景 | 说出 | 留住 | 活下去 |
| --- | --- | --- | --- |
| 恋爱开始 | 送他一首诗 | 主动吻他 | 先行离开 |
| 是否告发同伴 | 只承认自己：承担自己的部分，不提供同伴姓名 | 拒绝告发：一个名字也不说 | 说出名字：希望换取减轻处理，但结果未改变 |
| 最后一夜 | 把卡姆兰、婚姻、机票、害怕与爱一次说完 | 先说清卡姆兰，再问阿拉什是否愿意寻找另一条路 | 只说航班已确定，把卡姆兰的名字留在信封里 |

第二次选择前只允许玩家知道：玛兹雅当夜被带走、去向不明；学校追问谁参与了刊物。她六个月后获释和后续生活只能在选择后揭示。玩家可只承认自己、拒绝告发或说出名字；三条路线都不能被写成道德标准答案。

第三次选择的三个选项都要明确莱拉是否、何时、以何种方式提到卡姆兰。无论选择哪一项，两人都在机场分开。

### 2.2 三次共鸣操作

共鸣不计入三轴得分，但会改变措辞、镜头、物件、声音与结尾组合。它们不能再复用普通三选一卡片。

#### 照片：放置

- 桌面上存在“正面朝上”“反面朝上”“收回包里”三个可见放置区域。
- 支持鼠标拖拽、触摸/指针拖放，以及“先选照片、再点目标区”。
- 放置后必须看到照片落桌、翻面或滑入包的画面状态，再显示感官确认。
- 提供键盘与读屏等价按钮；拖拽不是完成流程的唯一方法。
- 终章诗集与咖啡馆中的毕业照必须来自同一张 canonical asset，同一人物、站位和裁切来源保持连续。

#### 邮件：写入并删除

- 使用真实的邮件编辑器语义，玩家从三句作者预设文本中选一句写入输入框。
- 玩家通过一次明确点击删除草稿；按钮原生支持触摸、鼠标、Enter 与空格。
- 删除后仍须呈现“写入 → 已删除”的状态变化；减少动态模式只减少动画，不省略文本。
- 不开放自由文本分类，不用模型推测玩家意图。

#### 视线：移动镜头

- “手与白发”“发黄的诗集”“时钟与机场方向”是重逢画面上的三个视觉热点。
- 悬停、触摸或键盘聚焦时，焦点、遮罩、裁切或景深真实转移到对应目标。
- 确认后，所选焦点决定后续镜头语言；下一幕不能机械重复同一句提示。
- 提供读屏可访问的等价按钮或列表，但可见主操作仍发生在画面上。

六次输入至少形成三种不同的操作语法：选择卡、物件放置、持续删除与画面热点。玩家不应把体验概括为“点六次三选一”。

第一次主选择前另有两步不计分的放映机协作：莱拉先扶住片门，再与阿拉什收紧胶片、点亮灯泡。它只负责建立手感与亲密，不成为隐藏的第七次测评。

## 3. 记忆回响契约与路由

主选择数据至少拆分为：

- `action`：当场发生的动作；
- `confirmation`：当场感官确认；
- `nearEcho`：一至两幕内的近期后效；
- `farEcho`：跨年代回响；
- `endingFragment`：基于该具体动作的结尾镜头；
- `revisitEcho`：剪辑室的新旧未来对照。

共鸣数据至少拆分 `action`、`confirmation`、`farEcho` 与 `endingFragment`。字段不能互相代用，以免确认句在后续场景原样重播。

远期路由应配置在 `futureEchoRoutes`，而不是散落为 JSX 条件：

| 未来场景 | 最多追加的动态回响 |
| --- | --- |
| 咖啡馆到达 / 重逢视线 `gaze` | 第一次主选择、邮件 |
| 诗集 `book` | 照片、第二次主选择 |
| 路口 `crossroads` | 第三次主选择 |

视线只改变最终镜头，不在下一幕重复成一条回响。任一未来场景最多追加两条动态回响；同一句 `confirmation`、`nearEcho` 或 `farEcho` 不得在相邻场景重复。

## 4. 结尾不是测评面板

完成选择后，先播放不超过 120 个汉字的个性化电影尾声，再允许玩家展开档案。电影尾声由以下四部分组合：

1. 主导动作轴决定总体语气；
2. 次轴提供反题，承认少数派选择；
3. 一件实际选择过的共鸣物件进入镜头；
4. 现实生活共同收束：卡姆兰与玛丽亚姆在所有路线中都明确存在。

混合尾声不暗示“最完整”或“真结局”，基调句为：

> 她没有让一种记忆替另外两种作证。

“本轮主调、三段主记忆、三段共鸣、未选择残片、前后对照”等信息放入默认收起的“查看本轮剪辑 / 记忆档案”。情绪尾声必须先完成，元数据不能抢先出现。

共享现实收束可围绕以下动作润色：莱拉过街后把航班时间发给卡姆兰；阿拉什回拨玛丽亚姆，询问今晚的云会不会遮住流星。旧事没有消失，眼前生活也没有暂停。

## 5. 首轮与记忆剪辑室

### 首轮

- 隐藏开发版本、`01/25` 场次感、主导轴标签和精确分数。
- 自动保存继续工作，手动保存不作为常驻主按钮。
- 记忆册只展示物件、玩家获得的记忆句和模糊的“记忆天气”。
- 标题页提供“有声进入 / 静音进入”或等价明确入口，音频只能由用户手势启动。

### 通关后

- 解锁说出 / 留住 / 活下去与其对应的理想、亲密、生存解释。
- `快速重剪`：只重做三次主动作，照片、邮件与视线沿用上轮。
- `完整重剪`：六次输入全部可重做。
- 每次改选后先显示“上轮回响 → 本轮回响”，再进入下一处剪辑。
- 最终比较默认只展示发生变化的片段；沿用内容可以折叠，不重复堆叠。
- 未选择的记忆残片保持克制，不制作成奖杯墙或“全结局收集”。

## 6. 可访问性、触控与减少动态

- 所有点击目标至少 `44 × 44px`，390 × 844 下不出现横向滚动。
- 三种画面操作均能由触摸、键盘和读屏完成；不可只监听悬停或拖拽。
- 焦点样式可见，弹窗锁定背景滚动，关闭后焦点返回触发器，桌面端支持 Escape。
- 选择与共鸣场景不能被空格、回车或连续点击跳过。
- `prefers-reduced-motion` 只减少位移、闪光和等待动画，不压缩阅读时间，也不跳过确认状态。
- 邮件删除、照片放置和视线聚焦均通过 `aria-live` 或等价状态向读屏用户反馈。

## 7. 声音入口与叙事声音

- 默认静音；标题页提供“有声进入 / 静音进入”，首次用户手势后才创建或恢复音频上下文。
- 声音偏好保存在本设备，切换、重剪和组件卸载时正确停止节点与定时器。
- 优先使用放映机马达、荧光灯、电邮删除、机场模糊广播、茶杯与雨声等叙事性环境声。
- 照片、诗页、灰烬、车票拥有克制的声音母题，并在远期回响或结尾复现。
- 音乐只作为低干扰氛围；未经文化与音乐复核，不宣称“严格传统波斯音乐还原”。

## 8. 静态验收清单

- 三个物理交互组件独立存在：`PhotoInteraction`、`EmailInteraction`、`GazeInteraction`。
- `futureEchoRoutes` 中每个场景最多两条动态回响，且路由符合第 3 节。
- `composeCinematicEpilogue` 负责组合不超过 120 汉字的电影尾声。
- 第二次选择前没有“六个月后获释”的剧透；该信息在选择后出现。
- 第三次主选择的三个选项都明确如何处理卡姆兰。
- 首轮不显示轴向与精确分数；通关后档案可以解释轴向。
- 结尾档案默认折叠，所有路线的电影尾声都包含卡姆兰与玛丽亚姆的现实存在。
- 三处毕业照镜头引用同一 canonical asset。
- 快速重剪与完整重剪均有明确入口，比较默认只列发生变化的片段。
- 1440 × 900 和 390 × 844 均可完成流程；触摸、键盘、读屏与减少动态均有等价路径。
