# 《革命街没有尽头》V3.1 专业试玩审计与执行提示词

> 历史执行记录：本文保留旧轮次的审计与提示词，不再定义当前玩家规则。现行叙事与玩法契约以 [`V5-DESIGN-CONTRACT.md`](./V5-DESIGN-CONTRACT.md) 为准。

## 试玩结论

V3 已经从“人格测试”跨过了最重要的一步：每个选择确实能在近期、远期和结局留下具体回声。当前主要问题不在故事是否成立，而在六次输入的操作语法仍然过于相似，界面把不同性质的行为都压成三张卡片；玩家理解了文本差异，却未必在手感上感到自己正在“摆照片、删邮件、移动视线”。

本轮浏览器自动点击受 Windows 登录策略 1385 阻断。确定性结论来自完整状态机、渲染代码、响应式样式和构建后测试；画面层级判断结合项目现有宽银幕素材，属于实现级视觉审计，不冒充截图实测。

## 调优建议

### P0：必须修复

1. 共鸣动作去问卷化：照片、邮件、视线应使用三套不同的卡面语言与操作提示，避免六次输入都像三选一测试。
2. 删除机制泄露：首轮确认中不出现“分数”“不改变分数”等字样；用动作和物件承担反馈。
3. 修复结局残片过滤：判断“未选择”时必须同时检查主选择和共鸣动作，不能把玩家写过的邮件误列为未选择。
4. 记忆册动态化：在不显示轴向分数的前提下，让“纸面／光线／远景”随已保存内容发生变化。
5. 结局信息分层：三次主选择是正文，三次共鸣是画外细节；避免六段同权文本造成阅读墙。

### P1：显著提升体验

1. 主界面用“第几幕”代替醒目的百分比，把产品气质从网页进度条拉回电影叙事。
2. 近期回声场景增加一个克制的“物件回声签名”，让玩家不用翻记忆册也能看见选择已抵达未来。
3. 章节与地点继续复用大图，但通过不同裁切、物件层和色温区分场所，减少同图跨地点造成的舞台感。
4. 移动端强化底部安全区、确认卡和三类共鸣卡的可读性；保持单列且不横向滚动。
5. 记忆册补完整焦点循环，保证键盘和屏幕阅读器用户不会跳到背景。

### P2：后续内容制作

1. 为照片、名单、邮件、诗集各补一张真正的局部近景图，替换当前 CSS 抽象物件框。
2. 为每章补 20–35 秒可无缝循环的实录环境层：雨棚、荧光灯、远处交通、机场大厅、咖啡馆杯碟。
3. 增加一次“无选项的物件操作”：玩家亲手拖动照片或按住删除键，但仍不改变历史结果。
4. 若扩充到正式版，把每章设计为“输入—近期回声—代价镜头”的固定节拍，而不是增加更多分支。

## 可执行提示词

### Prompt 01：共鸣动作界面分化

在不改变数据结果的前提下，为 photo、email、gaze 三类 resonance 增加独立样式和动作引导。照片像桌面摆放位，邮件像三条未发送草稿，视线像三个镜头焦点。保留 44px 触控区、键盘焦点和移动端单列。验收：DOM 有三类专属 class；视觉文案不出现分数；三类操作仍写入同一共鸣记录。

### Prompt 02：文学化确认与回声签名

给 9 个共鸣选项增加配置化 confirmation。选择后使用具体动作句，不解释系统。近期 echo 场景显示“回声抵达 · 物件”签名。验收：所有共鸣 confirmation 非空；页面不包含“不改变分数”；三个 echo 场景均能显示所选 motif。

### Prompt 03：动态记忆天气

记忆册首轮只展示“纸面／光线／远景”三个模糊状态，状态随已保存的主选择与共鸣数量、组合变化，但不显示轴向名称和精确点数。通关后才显示说出／留住／活下去解释。验收：0、1–2、3 个主记忆和不同共鸣数量会得到不同天气文案；首轮视觉无分数。

### Prompt 04：组合结局重新分层

把三条主选择 endingFragment 放入“亲手留下的三段记忆”，把三条共鸣 endingFragment 放入“画外仍在发生”次级区域。修复未选择残片过滤，使 answers 与 resonances 的 optionId 都参与排除。验收：结局不出现已选择的“未选择残片”；正文和共鸣有明确视觉层级。

### Prompt 05：电影化进度与移动端收口

顶部从百分比改为“第 08 / 25 幕”，进度条保留但提供可访问标签。为移动端加入安全区、共鸣卡专属布局和确认层防遮挡。验收：360×640 不横向滚动；所有控制至少 44×44；进度不再像属性面板。

### Prompt 06：测试与验收

增加结构测试覆盖共鸣确认、三类共鸣样式、动态记忆天气、结局分层和残片过滤；执行构建、全部测试、启动器检查以及首轮＋重访状态机回归。验收：构建成功、测试全绿、四类结局可达、重访差异仍显示具体未来回声。
