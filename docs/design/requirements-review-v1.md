## 2. 六个核心决策

### 决策 1：纵切片三场景的"行为可玩性"门槛

**决策内容**：

- 每个场景必须支持 ≥ **6 种**结构化行为（从 12 种里挑）
- 每个场景至少 **1 次不可撤回**干预（destroy / conceal / give / promise 之一）
- "AI 原生门槛" = 两条都满足：
  - **A 触达**：至少 1 条玩家早期行为，**在未来场景被 NPC 主动提起**（不是回声台账，是 NPC 在对话里说）
  - **B 因果**：至少 1 条玩家行为**改变了世界状态**（不是只改认知）

**反向代价**：门槛太低 → "包装过的下一页"重现，AI 原生名不副实。

**验收标准**：
- 每个新增互动提交时，必须由 `four-questions-guard.py` 自动跑 A/B 两条
- 任意一条不通过 → 阻断提交，要求修改

---

### 决策 2："记忆修复者"的具体体感

**决策内容**：

- **默认 = 第三人称旁观者**
- 视角切换 = **付费解锁**（¥3/段 或 收藏版包内）
- 旁白用"你看到了 X"而不是"你做了 X"——保持距离感
- 解锁视角后，**不是换 UI，是换叙事**（记忆账本、情绪演出、NPC 内心独白全变）
- 免费样章里**暗示**其他视角存在（"莱拉的角度是另一个故事"），但**不给**
- **# 补充条款（UP-20260715-015 / W11-B）**：
  视角付费解锁后**第一句台词必须引用 player 1985/2008 行为**——
  - 1985/2008 指的是 player 在更早年代的关键行为种子
  - 5 句备选台词对应 5 个跨年代行为种子，按 priority 升序取最触动
  - 落地于 3 个场景的 mandatory echo 统一设计模式：
    - `photo_lab_2008.admit_2008_behaviors_5_candidate_lines`
    - `farewell_2011.admit_1985_behaviors_5_candidate_lines`
    - `reunion_2024.first_words_admit_2008_2011`
  - 候选结构（narrative_contract.schema.json 已加 `candidate_lines`）：
    `{ line_id, text, speaker, seed_id, priority }`（+ era-anchored aliases）
  - selection_rule：NPC Agent 用 `referencedMemoryIds` 自动匹配 causal_seed，
    按 priority 取最触动；无任何 candidate 匹配 → line_01 兜底（**禁自由发挥**）

**反向代价**：
- 默认多视角 → 代入混乱、情感稀释、复购率掉
- 视角只换 UI → 玩家骂"骗钱"，伤害长期口碑
- 第一句台词自由发挥 → 玩家感受不到"跨年代回响"的重量，付费动机被稀释

**验收标准**：
- 免费样章完成时，调研回访中能听到"我想看莱拉视角"的声音（埋下付费钩子）
- 视角切换后，**记忆账本内容、情绪演出文案、NPC 内心独白 3 项至少有 2 项发生实质变化**
- 视角付费解锁后第一句台词**100%** 命中 candidate_lines 之一（零自由发挥）
- 3 个场景的 5 句备选设计模式统一（结构、字段、selection_rule 一致）

### 决策 3：跨年代回响的强度阈值

**决策内容**：

- **mandatory echo + optional echo 双轨制**
- 每个场景必须配置 mandatory echo，optional echo 可选

| 场景 | mandatory（必触发） | optional（条件触发） |
|---|---:|---:|
| photo_lab_2008 | 2（照片归属 + 名单处置） | 1 |
| farewell_2011 | 2（**必须引用 2008**） | 1 |
| reunion_2024 | 3（**必须引用 2008 + 2011**） | 1 |

- mandatory echo 只要玩家做了对应行为，**未来场景必触发**
- optional echo 只在特定组合下触发，最多 1 条
- 70%+ 测试玩家看到"早期 → 未来"回响的承诺——mandatory 设计下几乎必达成

**反向代价**：
- mandatory 太少 → 因果靠运气，玩家不会把"我做的事"和"未来发生的事"建立因果
- mandatory 太多 → 玩家觉得"太巧了"，沉浸感破
- optional 太多 → 模型不确定性大，玩家体验两极化

**验收标准**：
- 每个场景的 `narrative_contract.yaml` 必须显式声明 mandatory echo 列表
- 任何 NPC 主动提起的回响，**必须在 mandatory 列表里**——AI 导演不能自由发挥

### 决策 4：商业化档位（对外可分享版）

**决策内容**：

| 商品 | 价位 | 包含 |
|---|---:|---|
| 免费样章 | ¥0 | 序章 + 三场景缩略（每场 5 分钟）+ 1 次 mandatory echo |
| 案件通行证 | **¥25** | 纵切片完整三场景 + 2 次重演 + 200 积分 |
| 收藏版 | **¥48** | 完整三场景 + 双视角 + 原声 + 私人终章 + 500 积分 |
| 平行演算包 | ¥12 | 5 次额外重演 |
| 积分包 | ¥12 | 150 次主调用 |
| 额外人物视角 | ¥3 | 1 段 |
| 私人纪念品 | ¥8 | 本局专属信件 + 照片 + 关系报告（可导出） |

- **BYOK = P1，本版不做**
  - 理由：单局 ¥0.8 + 200 积分够 10+ 局，BYOK 应该是真实 token 焦虑投诉后才做的 P1
- 永久禁止：
  - 一句话说到一半要付费
  - 暗示付费 = 正确结局
  - 伪造限时、库存、外部品牌奖励
  - 剧情中途耗尽积分（必须降级而不是断线）
  - 模型自决价格

**反向代价**：
- 价位激进 → 转化率崩
- 价位保守 → 单位价值感低，玩家会嫌"AI 游戏就这价？"
- ¥25 的依据：AI 原生游戏的单位价值不是"看完整故事"，是"可玩性 × 因果深度"

**验收标准**：
- 每个付费点必须从"已结束 / 已解锁"状态触发，**不在主线中段**
- 收藏版的"私人终章"必须根据本局时间线生成，**不能是固定模板**
- BYOK 接口预留，但实现推迟到 v2

### 决策 5：AI 成本控制红线

**决策内容**：

**硬红线**（违反即重构）：

| 指标 | 红线值 |
|---|---|
| 30-45 分钟纵切片主调用次数 | ≤ **20 次** |
| 单次输出 token | < **800** |
| 单回合模型调用次数 | ≤ **2 次** |
| 关键交互响应 P95 | < **4s** |

**软目标**（可超 20%）：

- 单局 AI 成本 < ¥0.8
- 收入 8-12% 投入 AI（前期 20% 可）

**4 级降级链**（按优先级）：

1. **L1**：NPC 反应超时 → 用策划兜底台词
2. **L2**：Director 超时 → 跳过节拍校验，只跑 NPC Proposal
3. **L3**：Resolver 之前任何一步二次失败 → 主线走策划脚本（不调 LLM）
4. **L4**：Resolver 写库失败 → 弹"服务暂不可用"+ 保留存档

**反向代价**：
- 单级降级太粗 → 体验断崖，玩家觉得"游戏坏了"
- 4 级阶梯让玩家感觉"游戏在变通"

**验收标准**：
- 每次模型调用必须在 `model_calls` 表留痕（延迟、token、成本、降级状态）
- 连续 3 局触发 L3 降级 → 触发 P0 报警（说明 prompt 或合同有问题）
- 任何硬红线被突破 → CI 阻断

---

### 决策 6：四项问题自检落成工具

**决策内容**：

**落成 CLI + 嵌入 CI**，不是 checklist，是硬工具：

- `tools/four-questions-guard.py`（Python）
- 输入：互动 YAML/JSON
- 输出：4 项 yes/no + 描述性 + 阻断/通过
- 嵌入 `content-studio`：策划提交新场景时自动跑
- 嵌入 CI：每次 PR 改 `content/**/scenes/*` 触发
- 必含 4 项核心检查：
  1. 是否改变世界状态（artifact 归属 / event 账本）？
  2. 是否改变人物认知（belief_matrix）？
  3. 是否改变后续可用行动（action 白名单 / turn_budget）？
  4. 是否产生未来回响（causal_seed → 未来 scene）？
- 附加检查：
  - 是否有 forbidden_reveal 风险
  - 是否影响 turn_budget（会不会被这个互动打爆预算）
  - 是否影响 artifact 唯一性

**反向代价**：
- 只做 checklist → 走过场，违规互动照常上线
- 工具但不入 CI → 工具被绕过，等于没有原则

**验收标准**：
- 工具代码必须 100% 单测覆盖
- 工具运行失败 → CI 阻断 PR
- 工具输出必须有可读解释（不只是 yes/no）

## 3. 跨决策约束

6 个决策**互不冲突**，但有以下**隐含的协同约束**：

1. **决策 1 + 决策 3 必须配套**：
   - mandatory echo 是"AI 原生门槛 A 触达"的唯一实现路径
   - 每个 mandatory echo 必须在 `narrative_contract.yaml` 里登记

2. **决策 2 + 决策 4 必须配套**：
   - 默认旁观者 + 视角付费 = 商业化的核心支点
   - 视角解锁后必须**产生新的 mandatory echo**（不是 UI 变化）

3. **决策 5 + 决策 6 必须配套**：
   - 4 项自检工具的输出必须能反馈到 `model_calls` 表
   - 任何"自检通过但降级"的互动必须人工 review

4. **决策 4 + 决策 5 的成本边界**：
   - 案件通行证 ¥25 × AI 成本 8-12% = ¥2.0-3.0/局可用 AI 预算
   - 当前 20 次调用 / ¥0.8 单局 = ¥0.04/次，距离上限还有空间
   - 警示：BYOK 推迟 = 玩家无法自购更多 AI 算力，必须用 200 积分控好预期

## 4. 风险与预警

| 决策 | 错的话会怎样 | 监测点 |
|---|---|---|
| 1 门槛低 | "也就那样"，AI 原生名不副实 | 封测问卷：是否感受到"行为改变未来" |
| 2 默认多视角 | 情感断档、复购掉 | 复购率、首日回访率 |
| 3 mandatory 太多 | "太巧了"，沉浸感破 | 玩家访谈：是否觉得"刻意" |
| 4 价位激进 | 转化崩；保守则单位价值感低 | 转化漏斗、单 ARPU |
| 5 红线太严 | 降级频繁体验差；太松成本失控 | L1-L4 降级频次、成本占比 |
| 6 工具不入 CI | 自检失效 = 没有原则 | CI 拦截率 |
