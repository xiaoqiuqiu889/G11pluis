## 1. 背景

需求评审 2026-07-15 08:00 第 5 轮 cron 报告提出 **UP-20260715-024 critical**：

> **建议**：两案主角是"互为回忆中的对方"（第一案 leila 在 1985 莫斯科遇到的是 natasha 时代的 ilya；第二案 natasha 在 1985 遇到的 ilya 是第一案 arash 的前身）
>
> 原因：玩家从第一案（"我是 leila 的旁观者"）进入第二案（"我是谁？"），需要明确"两案主角 + 是否互为回忆中的对方"才能继续
>
> 这是 W12 启动的**前置 critical**

## 2. 评审推荐与 W6 已定稿设计的冲突

### 2.1 W6 已定稿的 case_02 设计（`anchors.yaml`）

```yaml
# anchors.yaml 顶部明确写：
# 时代：1985—2008，苏联晚期与解体后的中欧
# 关系：俄罗斯大提琴手（女主） + 犹太裔钢琴家（男主，1989 移民维也纳）
# 宏观端点：
#   - 女主因《这里没有童话》被苏联文化部"内部警告"
#   - 1989 男主犹太裔钢琴家携妻移民维也纳
#   - 女主 1993 嫁本地剧作家 Sasha / 男主 1992 娶维也纳指挥助手 Lisa
#   - 2008 柏林独奏会后后台
```

### 2.2 W6 已定稿的 case_01 设计（`anchors.yaml`）

case_01 主角 leila / arash 1985 年时是**德黑兰大学**文学系 + 物理系（21-22 岁），地点**完全在德黑兰**，**不会出现在莫斯科音乐学院 305 琴房**。

### 2.3 冲突点

| 评审推荐 | W6 已定稿 |
|---|---|
| 第一案 leila 1985 莫斯科遇到 ilya | leila 1985 在德黑兰 |
| 第二案 natasha 1985 遇到 ilya 是 arash 的前身 | natasha 1985 遇到的是 23 岁犹太裔钢琴家伊利亚 |
| 两案主角"互为回忆中的对方" | 两案主角是**完全独立的人物**（不同时代 / 不同地点 / 不同国籍）|

## 3. 决策

### 3.1 采纳方案：保留 W6 已定稿的"两案独立主角"

**两案主角独立**：
- case_01 leila / arash = 2008-2011-2024 德黑兰 → 圣何塞 → 伊斯坦布尔
- case_02 natasha / ilya = 1985-1989-2008 莫斯科音乐学院 → 维也纳 → 柏林

**跨案关系**：
- 两案**不共享主角**
- 两案**不共享宏观端点**（处分 / 离开 / 结婚 / 重逢 都是各自独立的事件）
- 两案**共享母题结构**（V5 命题"内容可规模化"实证）—— 同结构但不同事件 / 不同物件 / 不同声音
- 跨案母题对应表见 `content/_shared/motifs/cross_case_motifs.md`

### 3.2 不采纳 UP-20260715-024 的理由

| 理由 | 说明 |
|---|---|
| W6 已经定稿 | case_02 5 锚点 + 4 人物 + 3 scene 已经全部落盘并通过 W6 验证（UP-013 reunion 蒙太奇实现 / W11-B 5 句备选统一 / W11-A 美术声音）|
| 重大剧情变更 | 改"两案主角互为回忆中的对方"需要重写 5 锚点 + 4 人物 + 3 scene 全部 13 个文件 + 4 character 卡 + 5 anchor YAML |
| 跨案年代不匹配 | leila / arash 是伊朗德黑兰大学，natasha / ilya 是苏联莫斯科音乐学院——**地点 + 国籍 + 文化 + 时代完全不同** |
| V5 命题"内容可规模化"的另一条路 | "同结构但独立主角" 也是 V5 命题的一种实现——不要求跨案主角共享，而要求跨案**结构 + 决策红线 + mandatory echo 双轨制** 共享 |
| 风险高 | 大剧情变更需要用户拍板，不能由综合开发任务擅自决定 |

### 3.3 评审推荐的可能益处

我们承认评审推荐的可能益处（**留给 W13 复盘**）：

- **更深的跨案情感共鸣**：玩家从第一案到第二案感到"我一直以为在看别人的故事，其实我自己就是那个被看的人"——这种"跨案主角一体"的设计意图是有价值的
- **降低"我是谁"的认知负担**：玩家在第二案不需要重新建立认同

但**这些益处需要用户明确拍板**才能重写。**综合开发任务不擅自决定**。
