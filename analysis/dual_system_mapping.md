# v6 ↔ AI 原生双系统对照表

> 范围:本对照表覆盖《革命街没有尽头》现有 v6 系统(下文"v6")与 v0.1 PRD 描述的 AI 原生目标系统(下文"AI 原生")。
> 目的:为重构提供"逐项映射 + 改造/废弃/保留判定 + 优先级"的可执行蓝图。
> 方法:严格基于 v3.1 经验教训的"四项问题"自检——是否改变世界状态、是否改变人物认知、是否改变后续可用行动、是否产生未来回响。
> 经验教训关键词:"人格测试"、"包装过的下一页"、"公开爱情值"、"操作语法过于相似"、"没有让一种记忆替另外两种作证"。

## 1. 顶层设计理念对照

| 维度 | v6(现状) | AI 原生(目标) | 迁移策略 |
|---|---|---|---|
| **玩家身份** | 旁观书写者(v5/V6 契约:"玩家书写的是具体动作、保存的物件、看见的细节和记忆措辞")。在三个固定选择点替莱拉落字,在三个共鸣点替她摆放物件。镜头之外的人。 | **记忆修复者**——一个有名有姓、有动机的"在场的我"。能进入不同人物的视角(莱拉、阿拉什、卡姆兰、玛丽亚姆、玛兹雅),通过"恢复被压抑/被删改/被遗忘的真相"推动叙事。 | P0 重构。玩家身份必须从"代写者"升级为"修复者",否则"信念矩阵"无法激活——因为只有主动追问(Investigate)和揭示(Reveal)才会改变人物的"知道的事"。 |

| **商业化** | **截句式内购**:¥1 当前对白续接(只解一句)、¥2.9 本章完整对话、¥9.9 五章完整合集。**配合五枚 G1N-DEMO-* 京东码模拟礼包**。模拟运行、不扣款、不跨设备。 | **案例通行证 + 平行演算包**:免费样章作为钩子,案件通行证(¥18-25)解锁完整主线 + 信念卷宗;**平行演算包**(¥8-12)提供 3-5 次反事实时间线;人物视角包(¥12-18)解锁他人视角章节;纪念品(¥38-48 收藏版)实体化母题物件。 | P0 全部废弃。v6 的 ¥1/¥2.9/¥9.9 三档 + G1N-DEMO-2008-01 等五枚演示码必须**整体废弃**,详见 §7。 |

### 1.1 三句话总结差异

1. **v6 是"看片 + 填空"**:玩家代莱拉落字、摆物件、移动视线,宏观端点锁定,微观动作由玩家书写。**AI 原生是"修复 + 重构"**:玩家在信念矩阵中追问、揭示、安抚、告别,改变的不是"被记住的方式",而是"知道的事实"。
2. **v6 的因果是"装饰性的"**:`farEcho` 是文本字段,玩家改选后只是看到不同的话;**AI 原生的因果是"操作性的"**:玩家的行为直接修改 `beliefNode` 状态,后续 12 行为对同一节点产生不同可用性。
3. **v6 的玩家是"唯一叙事者"**:**AI 原生的玩家是"多个叙事的交汇点"**——同一段往事,莱拉视角、阿拉什视角、卡姆兰视角的"真相"是不同的;玩家必须通过 reveal / conceal / question 在人物之间周旋。

## 2. v6 三轴 → AI 原生信念矩阵

### 2.1 v6 的"人格测试"本质(v3.1 经验教训第 2 条)

v6 核心数据结构 `Scores = { speak: number, keep: number, survive: number }`(`game-logic.ts: emptyScores()`),`scoreAnswers()` 只是把 `AnswerRecord.axis` 字段做 `+1` 累加。**结果是"理想 vs 爱情 vs 生存"三维人格画像**——这正是 v3.1 经验教训里点名的"人格测试":

> 玩家理解了文本差异,却未必在手感上感到自己正在"摆照片、删邮件、移动视线"。——V3.1 P0 第 1 条

- **speak 轴** = "我把最难的话说完" → 测量"坦率人格"
- **keep 轴** = "我留住一个吻、一本书和共同未来" → 测量"依恋人格"
- **survive 轴** = "我先确认出口" → 测量"务实人格"

更致命的是 v5/V6 的 `axisExplanations` 在通关后会显式解释:"它后来被称作理想 / 爱情 / 生存"——**这正是经验教训第 4 条"公开'爱情值'问题"**:当玩家被告知"刚才的选择代表了你对爱情的看法",它就从"叙事动作"退化成"心理测试题"。

### 2.2 三个轴在叙事中实际上是什么

剥开"人格测试"包装,三次主动作本质上在改变**人物知道的事**和**后续可用行动**:

| v6 轴 | v6 表面动作 | 实际改变的"世界状态" | 实际改变的"人物认知" | 实际改变的"后续行动" |
|---|---|---|---|---|
| **说出 (speak)** | 送他一首诗 / 只承认自己 / 一次说完 | 莱拉把内心曝光给阿拉什(信任) | 阿拉什知道"她愿意让我看见脆弱面" | 后续停电场景,他会先问"你还好吗"再靠近 |
| **留住 (keep)** | 主动吻他 / 拒绝告发 / 先说卡姆兰再问另一条路 | 关系契约形态被塑形(从约会到共同生活到分手边界) | 莱拉把"卡姆兰"放到谈判桌上 | 机场场景里,告别台词与松手时机会不同 |
| **活下去 (survive)** | 先行离开 / 说出名字 / 只说航班 | 离开的"理由叙事"被改写(理想 / 妥协 / 效率) | 阿拉什被告知"她离开的逻辑" | 2011-2023 蒙太奇里,旧爱的重量不同 |

**关键洞察**:三轴的 *叙事功能* 已经在那里——它改变人物知道什么、决定后续如何告别。但 v6 把这层叙事功能**抽离成数字字段**,再用 `axisNames / axisExplanations` 让玩家反过来用"理想 / 爱情 / 生存"去解释自己的选择——这是一个**自我挫败的回路**。

### 2.3 AI 原生如何改为"关系模拟"

AI 原生的信念矩阵是**四层而不是三维**,且**不是"玩家人格画像",而是"人物/玩家在每个节点持有的认知状态"**:

```
beliefMatrix[nodeId][characterId] = {
  objective:    "客观事实",     // 不依赖任何人认知的事实
  character:    "人物认知",     // 该人物相信的事
  memory:       "人物记忆",     // 该人物记住的事(可能失真)
  secret:       "隐藏秘密",     // 该人物知道但未说出口的
}
```

| v6 三轴 | AI 原生信念矩阵层 | 改造方式 |
|---|---|---|
| **speak(说出)** | `secret → character / objective` 的 reveal 边 | "她把最难的话说完" = 把 `secret` 推到 `character` 公开层;在机场告别时,这个 reveal 会让阿拉什在重逢时改变他的开场白。**不再是 +1,而是"open secret"。** |
| **keep(留住)** | `objective → memory` 的 retain 边 | "我留住一页纸" = 让一个客观物件进入人物的 memory 层;在两座城市蒙太奇里,这个 memory 会在旧爱被同时想起时,触发"两人都保留着同一段记忆"的具体画面。**不再是 +1,而是"tagged memory"。** |
| **survive(活下去)** | `character → objective` 的 decide 边 | "我先确认出口" = 让一个行动目标被推到客观行为层;在重逢场景,这个决定会决定莱拉是否愿意停下脚步听阿拉什说"我到了"。**不再是 +1,而是"actor state change"。** |
| **评分制(每轴 +1)** | **废弃**。AI 原生不计算"玩家属于哪一型",只计算"此时此地,每个人物知道什么、能做什么、愿意做什么"。**`emptyScores()` 整个函数必须废弃。** | 把"决定性测评"改为"持续状态查询"——玩家在每个节点问的是"我现在能做什么",不是"我是哪种人"。 |

### 2.4 关键改造点

- **`scoreAnswers()`、`determineEnding()`、`determineSecondaryAxis()` 全部废弃**。`mixed` 结局是"三轴同分时回到混合"的退路机制,在 AI 原生里没有对应物——AI 原生没有"哪一种人格占主导"的问题,只有"哪一种关系在当下是活的"的问题。
- **V5 端点契约"宏观端点锁定"必须保留**——莱拉遭到处分 / 决定离开 / 与卡姆兰结婚 / 十三年后与阿拉什重逢 / 路口再次分开——这是叙事的脚手架,AI 原生只是把"微观动作"从三选一升级到 12 行为。
- **"她没有让一种记忆替另外两种作证"必须保留**(V5 第 4 节基调句),但实现方式改变:不再靠 `axisPairCounterpoints` 表查找,而是让四层信念矩阵之间**永远存在未对齐的张力**——玩家在任一节点都看到同一段往事有 N 种合法解读,系统不强制收束。

## 3. v6 6 输入 → AI 原生 12 行为映射

v6 6 次输入由 `interactionCatalog` / `memoryContracts` / `resonanceContracts` 定义;AI 原生 12 行为按"功能"(追问 / 揭示 / 隐藏 / 询问 / 对质 / 安慰 / 给予 / 毁坏 / 承诺 / 等待 / 离开 / 沉默)分类。

### 3.1 总览映射表

| v6 输入编号 | v6 行为类型 | v6 界面语法 | AI 原生 12 行为 | AI 原生补充 |
|---|---|---|---|---|
| **第 1 主选择** (恋爱开始) | 三选一卡片 | 三张并排卡:送诗 / 吻 / 离开 | **reveal + give + wait 的组合** | 送诗=reveal 内心 + give 一个文本;吻=give 身体接触;先离开=leave(以"主动"形式)。AI 原生把这三个动作拆开,允许玩家在同一个场景里 reveal 后再 wait(等对方反应) |
| **第 2 主选择** (是否告发) | 三选一卡片 | 只承认自己 / 拒绝告发 / 说出名字 | **conceal + reveal + confront 的对立** | "只承认自己"=conceal 同伴 + reveal 自己;"拒绝告发"=conceal 所有;"说出名字"=reveal 同伴(以交换自己减刑)。三选一被拆为 conceal/reveal 比例的连续选择 |
| **第 3 主选择** (最后一夜) | 三选一卡片 | 全说 / 先说卡姆兰再问另一条路 / 只说航班 | **promise + question + silence 的组合** | 全说=promise 一种关系(坦白一切);先说卡姆兰=question 阿拉什的边界;只说航班=silence 关于感情的承诺(只承诺操作信息) |
| **照片拖放** | 物件放置 | "正面朝上 / 反面朝上 / 收回包里"三区 | **reveal / conceal / give** | 正面=reveal 这段关系;反面=conceal 摄影日期;收回=give 这张照片给未来的自己。物件本身变成信念载体 |
| **邮件写删** | 物件操作 | 选三句预设之一 → 写入 → 删除 | **destroy + silence** | 写入=暂时 reveal 给服务器;删除=destroy 这个表达;玩家永远无法 re-undo——destroy 边是不可逆的 |
| **视线移动** | 热点聚焦 | "手与白发 / 诗集 / 时钟与机场方向" | **investigate / question / wait** | 选诗集=investigate 阿拉什这十三年;选手=question 他的手为什么变这样;选时钟=wait(等航班信息显示) |

### 3.2 v6 输入类型 → AI 原生语法的形式化映射

v6 真正有"操作语法多样性"的是三类共鸣(照片 / 邮件 / 视线),v3.1 经验教训 P0 第 1 条已经指出"六次输入都像三选一测试"——**问题不在内容,在界面语法**。AI 原生要把 6 输入彻底改为 12 行为的"操作语法族":

```
AI 原生操作语法族:
  ├── 物件类 (Photo/Email 升级)
  │     ├── 拖放    → reveal/conceal/give
  │     ├── 写删    → destroy/silence
  │     └── 组合    → confront/comfort
  ├── 空间类 (Gaze 升级)
  │     ├── 聚焦    → investigate/question
  │     ├── 移动    → wait/leave
  │     └── 翻面    → conceal/reveal
  ├── 声音类 (新增)
  │     ├── 提问    → question
  │     ├── 回答    → promise/silence
  │     └── 沉默    → silence/wait
  └── 关系类 (新增:可选择"与谁对话")
        ├── 莱拉视角   → 同样的 12 行为,作用于她自己的 secret
        ├── 阿拉什视角 → 同样的 12 行为,作用于他对她离开的认知
        ├── 卡姆兰视角 → 同样的 12 行为,作用于他对婚姻边界的理解
        ├── 玛丽亚姆   → 同样的 12 行为,作用于她自己的独立生活
        └── 玛兹雅     → 同样的 12 行为,作用于她对告发事件的记忆
```

### 3.3 必须保留的 v6 输入特征

- **三次主动作"先描述行为,不展示轴或分数"**(V5 第 2.1 节)→ AI 原生必须保留"首轮不暴露 beliefMatrix 内部数字"的等价原则。
- **三次共鸣操作"不能复用普通三选一卡片"** → AI 原生的 12 行为 UI 必须**至少四套不同的视觉语法**(物件 / 空间 / 声音 / 关系选择),不能全压成按钮。
- **第一次主选择前的"两阶段放映机协作"不计分** → AI 原生在玩家完成第一组 reveal/give 之前,所有 `beliefNode` 状态变化只进入 `memory` 层,不入 `objective / character` 层,避免"还没开始就被打分"。

## 4. v6 15 互动 → AI 原生四项问题自检

### 4.1 四项问题定义(来自 v3.1 经验教训)

| # | 问题 | v6 评估方式 | 失败信号 |
|---|---|---|---|
| 1 | 是否改变**世界状态**? | 物件是否被放置 / 邮件是否被删除 / 视线是否锁焦 | 玩家做了"看"动作,但世界没有任何持久变化 |
| 2 | 是否改变**人物知道的事**? | `farEcho` 字符串是否派发到未来场景的 `AnswerRecord.confirmation` | 玩家做了"问"动作,但人物认知层无变化 |
| 3 | 是否改变**后续可用行动**? | `availablePurchaseOffers` / 场景路由是否变化 | 玩家做了选择,但下一幕还是同样三个选项 |
| 4 | 是否产生**未来回响**? | `futureEchoRoutes` 是否在 2024 重逢 / 咖啡馆 / 路口触发 | 玩家在 2008 的选择,2011/2024 完全没影响 |

判定规则:
- **4 项都 ✅ → 保留**(v6 原型机制值得继承)
- **2-3 项 ✅ → 改造**(原型对了一半,需要补足缺失维度)
- **0-1 项 ✅ → 废弃**——这是经验教训里说的"**包装过的下一页**":表面上是互动,本质是翻页,做完不改变任何东西,只是把"下一段文字"解锁出来

### 4.2 15 个互动逐项自检

| # | v6 互动 | 1 改世界 | 2 改认知 | 3 改行动 | 4 改回响 | 判定 | AI 原生如何改造 |
|---|---|:-:|:-:|:-:|:-:|:-:|---|
| 1 | **photo-placement** (安放毕业照) | ✅ 照片状态(正面/反面/包里) | ✅ 阿拉什下一幕的台词 | ⚠️ 部分:照片/邮件/视线三共鸣在同一章只能改部分后续 | ✅ 第十三章重逢时,桌上哪张照片先被认出 | **保留+升级** | 升级为 reveal/conceal/give 三态,玩家在咖啡馆可以再选"把照片翻面给他看"。新增"将照片给卡姆兰看"分支——把物件升格为**关系传递载体** |
| 2 | **projector-repair** (修放映机) | ✅ 放映机齿轮状态 / 灯泡 | ✅ 阿拉什"我到了"台词 | ✅ 后续毕业照场景可重访 | ⚠️ 部分:主要影响 2008-2009 段落,跨年代弱 | **保留+补足** | 补足"放映机在 2024 重逢时是否还在"——把它从一次性协作升级为**跨年代物件母题**(详见 §6 重演系统) |
| 3 | **first-memory-action** (停电后的第一个动作) | ⚠️ 仅话语 | ✅ 阿拉什对她的认知 | ✅ 后续停电场景的开场 | ✅ 路口松手时机 | **保留** | 这一互动是 v6 三主选择的模板,转译为 AI 原生的 reveal + give + wait 组合(详见 §3.1) |

## 5. v6 显影度 → AI 原生信念状态

### 5.1 显影度 0-100 是什么

v6 `memoryExposure: number`(0-100)由 15 个互动 + 5 个章末奖励累加,各互动分别贡献 4-7 点,总互动 89 点 + 章末 15 点 = 104 → `clampMemory` 钳到 100。**5 档**(0-20 / 21-40 / 41-60 / 61-80 / 81-100)对应不同"解锁内容":

| 显影度范围 | 档位 | 解锁内容 |
|---|---|---|
| 0-20 | 基础剧情 | 25 幕主线 |
| 21-40 | 内心独白 | 角色内心片段 |
| 41-60 | 场景细节与纪念物 | 物件说明、组合线索、卷宗细节 |
| 61-80 | 隐藏对白预览 | 卷宗里看到未解锁对白入口 |
| 81-100 | 完整记忆卷宗 + 特别尾声 | 完整档案层级、特别尾声 |

### 5.2 v6 显影度的根本问题

显影度 0-100 是一个**单一标量**,把"玩家做了多少事"压缩成一个数字——这是经验教训第 4 条"公开'爱情值'问题"的延伸:

1. **它把"信念状态"量化成 XP**——21 / 21 / 20 / 19 / 19 这样的分布就是 RPG 经验值。V3.1 经验教训点名了"网页进度条"——"主界面用'第几幕'代替醒目的百分比"是 P1 任务,说明 v3 已经有这个气味。
2. **它向玩家公开**:`memoryTier(value)` 暴露 5 档,`nextMemoryUnlock(value)` 直接告诉玩家"距离下一档还差 5 点"——**这就是"公开爱情值"在数字层面的实现**。玩家点互动时第一反应是"这个值多少分",而不是"我会改变什么"。
3. **它奖励"做完所有事"而不是"做出有意义的差别"**:v6 的 15 互动做完 = 100,通关后所有隐藏对白可见。AI 原生要反过来——做完 15 互动可能只解锁 60% 的 beliefNode,因为某些节点**只在玩家选了特定行为组合后才进入可见状态**。

### 5.3 显影度增量(21/21/20/19/19=100)映射

| v6 显影度来源 | 贡献点数 | AI 原生对应物 |
|---|---|---|
| 第一章 3 互动 + 章末奖励(5+7+6+3) | 21 | 第一章 3 个 beliefNode:`photoNode` / `projectorNode` / `firstSpeakNode` + 章末信任节点 |
| 第二章 3 互动 + 章末(5+7+6+3) | 21 | 第二章 3 个 beliefNode:`publicationNode` / `namingNode` / `disciplineNode` + 章末处境节点 |
| 第三章 3 互动 + 章末(5+7+5+3) | 20 | 第三章 3 个 beliefNode:`departureNode` / `lastNightNode` / `airportNode` + 章末告别节点 |
| 第四章 3 互动 + 章末(5+7+4+3) | 19 | 第四章 3 个 beliefNode:`dualCityNode` / `emailNode` / `receiptNode` + 章末分离节点 |
| 第五章 3 互动 + 章末(5+6+5+3) | 19 | 第五章 3 个 beliefNode:`reunionNode` / `photoPairNode` / `crossroadNode` + 章末特别尾声节点 |

**关键差异**:v6 的 21+21+20+19+19 是**固定累加**(15 个互动做完 = 100),AI 原生必须改为**条件累加**:
- `photoNode` 完整解锁条件:在 2008 拍摄时同时完成 `reveal + give`(玩家必须选"我让他看我"且"我把照片给他")
- `namingNode` 完整解锁条件:玩家必须先 `investigate` 玛兹雅的去向,再 `conceal / reveal` 她的名字
- `lastNightNode` 完整解锁条件:必须 `promise` 至少一件事 + `silence` 至少一件事(只能"有说 + 有不说"才完整)

这样,**做完所有 15 互动 ≠ 100% beliefNode 可见**;某些节点需要特定的"行为组合"才能完整进入视野——这是经验教训第 1 条"行为组合"的具体实现。

### 5.4 公开"爱情值"问题 → AI 原生如何避免

v6 的失败模式有 4 种:

| 失败模式 | v6 表现 | AI 原生对策 |
|---|---|---|
| 数字可见 | `nextMemoryUnlock` 直接返回"还差 5 点" | **不显示数字,只显示"已解锁 / 未解锁"的二态标记**。玩家看到的是"档案里有 1 段未展开的对话",而不是"差 5 点升到 21 档" |
| 分档可见 | `memoryTier` 把 0-100 切成 5 档,档位名称有"内心独白 / 隐藏对白"等诱人字眼 | **不显示档位**。只显示"档案里有 8 段已记、4 段未记" |
| 进度条可见 | V3.1 经验教训点名"网页进度条" | 用"第 03 / 15 段记忆"代替百分比,底部只显示章节幕次 |
| 行为评分公开 | `axisValues` + `axisExplanations` 通关后解释"它后来被称作理想 / 爱情 / 生存" | **不解释行为的人格意义**。通关后只显示"在 2008 停电里,她选择了 X;在 2011 机场,她没有 Y"——历史叙述,不人格归类 |

**额外原则(AI 原生独有)**:
- **服务端可以计算 beliefMatrix 的所有数字,但客户端只渲染玩家被允许看到的字段**——付费墙、年龄分级、章节解锁都通过字段过滤实现,不是把数字藏起来。
- **数字在调试面板可见,但玩家永远不直接看到"我离满还差 N%"**——把"完成度"翻译成"她在这段过去里还留着几句话没说出口"这种叙事语言。

## 6. v6 重演系统 → AI 原生重演 + 平行演算

### 6.1 v6 现有重演

V5 第 5 节定义两种重演:
- **快速重剪**:只重做三次主动作,照片 / 邮件 / 视线沿用上轮
- **完整重剪**:六次输入全部可重做
- 比较时**默认只展示发生变化的片段**(避免重复堆叠)
- 未选择的记忆残片保持克制,不制作成奖杯墙

**v6 重演的局限**:
1. 只能"重做自己之前的输入",不能"假设别人做了不同选择"——这是单线回放,不是反事实。
2. 比较只在"自己 vs 自己"之间,无法看到"如果我当初告发,莱拉会怎样"。
3. 终点永远是 2024 路口分岔——"宏观端点锁定"虽然是叙事承诺,但也是商业限制。

### 6.2 AI 原生重演 + 平行演算

| 模式 | v6 现状 | AI 原生 | 商业化定位 |
|---|---|---|---|
| **重演(原"重剪")** | 快速重剪 / 完整重剪 | 保留,改进 UI:沿用 V5 原则——只显示发生变化的片段,加"新片段 vs 旧片段"对比 | 免费,基础权益 |
| **平行演算(新增)** | 不存在 | **3-5 次反事实时间线**——基于玩家本轮行为,服务端推演"如果莱拉在第 2 章选了不同行为,2024 路口会怎样"。玩家解锁的是**信念矩阵的反事实投影**,不是新故事 | 平行演算包 ¥8-12(详见 §7) |
| **人物视角(新增)** | 不存在 | 在主线之外,玩家可以进入卡姆兰 / 玛丽亚姆 / 玛兹雅视角,看到他们对同一段往事的不同 beliefNode | 人物视角包 ¥12-18 |
| **完整重访(原"完整重剪")** | 重做全部 6 输入 | 升级为"信念矩阵重置"——玩家可以选**重置哪些章节的哪些 beliefNode** | 基础权益(免费) |
| **章节回访** | V5 第 5 节 | 升级为"任意 beliefNode 跳转"——玩家可点击记忆卷宗中的任一节点,直接回到触发它的场景 | 基础权益(免费) |

### 6.3 平行演算的 3-5 次反事实时间线

**这是 AI 原生最有商业价值的创新**。服务端基于本轮 beliefMatrix 状态,推演 3-5 个"如果当时……"的分支:

| 反事实 | 玩家本轮做了什么 | 平行演算中"如果"什么 | 服务端如何生成 |
|---|---|---|---|
| **CF-1:告发的代价** | 玩家选择"拒绝告发" | "如果她在问话室说出了玛兹雅的名字" | 服务端把 `namingNode.secret` 反向 reveal,推演玛兹雅的 beliefNode → 推到她去设拉子后的"如果还认识莱拉"分支 |
| **CF-2:卡姆兰知情** | 玩家在最后一夜只说航班 | "如果她在最后一夜把卡姆兰全盘说出" | 服务端把 `lastNightNode.character` 层从"不知道"改为"知道",推演阿拉什在 2011-2023 蒙太奇里的行为变化 |
| **CF-3:重逢时不松手** | 玩家在路口选择"成熟地放开" | "如果他们在 2024 路口选择不放开" | 服务端把 `crossroadNode.wait` 时长拉满,推演"握住不松手"的剧本 |
| **CF-4:邮件没删除** | 玩家删除邮件 | "如果她没删除那封邮件" | 服务端把 `emailNode.destroy` 改为 `conceal`,推演邮件在 2011 圣何塞的生活里如何被卡姆兰看到 |
| **CF-5:照片翻面给卡姆兰** | 玩家把照片收进包里 | "如果她把照片给卡姆兰看" | 服务端把 `photoNode.give` 方向改为卡姆兰,推演卡姆兰的 secret 层变化 |

**关键设计原则**:
- **平行演算不重写主线**——三个宏观端点(处分 / 离开 / 重逢)始终成立,只是信念矩阵在分支内被改写。
- **平行演算不创造新剧情**——所有反事实都基于本轮已有的 beliefNode,服务端用因果图(`beliefGraph`)推演,玩家看到的是"推演结果",不是"平行世界小说"。
- **平行演算可分享**——玩家可生成"我的反事实时间线"链接,带权限控制(只读 / 同玩可解锁)——这是社交裂变的钩子。

## 8. v6 客户端权威 → AI 原生服务端权威

### 8.1 v6 客户端权威的事实

v6 `createInitialProfile(now)`, `normalizeProfile(input, now)`, `completeInteraction(profile, interactionId)`, `simulateLocalPurchase(profile, productId, now)`, `computeUpgradeCredit(profile, target)` **全部在浏览器执行**。`save-state.ts` 的 `CURRENT_CONTENT_REVISION = 1` 是客户端版本号,只用于兼容老存档。`save_resume` 事件在 V6 第 9 节里也只是"记录玩家点续玩的本地动作"。

**v6 客户端权威带来的 7 个具体问题**:

1. **结局被本地计算**:`determineEnding(scores)` / `composeCinematicEpilogue(endingKey, answers, resonances)` 在前端跑,玩家改浏览器存档即可解锁所有结局。
2. **付费权益被本地模拟**:`simulateLocalPurchase` 只改 `profile.entitlements`,不调支付 SDK,玩家改 `localStorage` 即可假装买了通行证。
3. **跨设备不同步**:v6 第 7 节"刷新后出现续玩入口"是单设备内的续玩,**没有云端存档**——换手机 = 重来。
4. **内容版本控制弱**:`CURRENT_CONTENT_REVISION = 1` 是常量,未来内容更新需要玩家手动迁移;无法做 A/B 测试、新剧情灰度发布。
5. **反作弊缺失**:玩家可以打开 console 改 `memoryExposure: 100`,所有 5 档解锁、`unlockedMilestoneIds`、`specialEpilogue: "available"` 全部瞬间到位。
6. **生产数据采集缺位**:V6 第 11 节"当前正式服务仍待接入,包括生产数据分析、用户同意、脱敏、留存、上报失败恢复和分析平台"——22 个事件只写本地调试日志。
7. **合规审查缺位**:未成年人保护、付费冷静期、内容分级、跨境数据流——全部待补。

### 8.2 AI 原生服务端权威

| 维度 | v6(客户端) | AI 原生(服务端) |
|---|---|---|
| **存档位置** | `localStorage`,`save_resume` 事件本地派发 | `/api/v1/saves/{userId}`,服务端持有**唯一权威存档**;客户端缓存只用于离线 / 弱网 |
| **结局计算** | `composeCinematicEpilogue` 客户端跑 | `/api/v1/epilogue`,服务端持有 `beliefGraph` 与叙事模板,客户端只渲染返回的 epilogue 字符串 |
| **付费权益** | `simulateLocalPurchase` 改本地 | `/api/v1/entitlements` 服务端验证,客户端只显示"已解锁 / 未解锁"二态;支付走 `/api/v1/checkout` 接 Apple IAP / Google Play / 微信支付 |
| **跨设备同步** | 不支持 | 支持,登录后服务端拉取最近存档;离线模式下客户端缓存,联网后 delta 同步 |
| **内容版本控制** | `CURRENT_CONTENT_REVISION = 1` 常量 | `/api/v1/content/revision` 服务端返回当前版本,客户端请求时携带客户端版本做兼容判断;支持灰度发布、A/B 测试、剧情分支实验 |
| **反作弊** | 无 | 服务端按用户 ID + 设备指纹 + 行为时间序列做异常检测;`memoryExposure` 不可能由客户端直接修改,只能通过行为流水结算 |
| **生产数据采集** | 22 事件只写本地 | 服务端按事件 → 主题流(Kafka / 内部等价物)→ 数据仓库(脱敏)→ 分析平台;用户同意(首次启动展示) / 撤回(设置页) / 跨境合规 |
| **合规审查** | 无 | 服务端做年龄分级校验(注册时 + 内容触发时双校验);付费冷静期;退款 API;内容分级标签 |

### 8.3 必须迁移的 v6 模块

| v6 模块 | 当前实现 | 迁移目的地 |
|---|---|---|
| `createInitialProfile` | 客户端默认值 | 服务端 `/api/v1/profile/init`,返回 `userId` + 初始 `beliefMatrix` |
| `normalizeProfile` | 客户端兼容老存档 | 服务端 `/api/v1/profile/migrate`,带 `clientVersion` 字段,服务端决定如何升级 |
| `completeInteraction` | 客户端累加 `memoryExposure` | 服务端 `/api/v1/behavior/record`,客户端只发"行为流水",服务端结算 `beliefNode` 变化 |

### 8.4 客户端可以保留的逻辑

不是所有东西都要上服务端。客户端仍负责:
- 渲染(场景画面、动画、音效)
- 输入采集(玩家点击 / 拖放 / 按住)
- 本地缓存(离线模式下最后一次服务端状态)
- 弱网优化(在网络抖动时不阻塞主循环)
- 视觉与可访问性(`prefers-reduced-motion`、键盘焦点、读屏标签)

**判定标准**:**任何会改变"玩家能看到什么内容 / 能做什么事 / 拥有什么权益"的逻辑,必须上服务端;只影响"画面怎么呈现"的逻辑,可保留在客户端**。

## 12. 总结:一句话差异

> **v6 是"看片 + 填空":玩家在锁定端点之间做 6 次结构化输入,数字决定他属于哪一型人;**
> **AI 原生是"修复 + 重构":玩家在 5 锚点之间做 12 种结构化行为,服务端按信念矩阵的 4 层状态结算他改变了什么、谁会知道、谁能继续行动、未来如何回响。**
> **保留 v6 的灵魂(人物、物件、母题、基调、端点、canonical 资产);废弃 v6 的病(人格测试、爱情值、包装过的下一页);用服务端权威与跨年代因果图,把"叙事动作"从文本字段升级为世界状态。**
